GitHub + Live Domain Auto Deploy Steps

1. Create a new empty GitHub repository.
2. In this project folder run:
   git init
   git branch -M main
   git add .
   git commit -m "Initial PHP hosting version"
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main

3. In GitHub repo open:
   Settings > Secrets and variables > Actions

4. Add these repository secrets:
   FTP_SERVER
   FTP_USERNAME
   FTP_PASSWORD
   FTP_TARGET_DIR

5. Secret values:
   FTP_SERVER: your hosting FTP host
   FTP_USERNAME: your hosting FTP username
   FTP_PASSWORD: your hosting FTP password
   FTP_TARGET_DIR: target folder on hosting, usually /htdocs/ or /public_html/

6. The workflow file already exists here:
   .github/workflows/deploy-php-hosting.yml

7. After secrets are added, every push to main will upload the public/ folder to hosting.

8. Important:
   - storage/rooms.json is excluded from deploy, so live room data will not be overwritten on every push.
   - If your hosting requires FTPS instead of FTP, change protocol in the workflow.

9. First live test:
   - Make a small text change in public/index.html
   - git add .
   - git commit -m "Test deploy"
   - git push
   - Open GitHub Actions tab and wait for green success
   - Refresh your live domain
